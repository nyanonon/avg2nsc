# avg2nsc

AVG32からNScripterへのシナリオデータコンバータです。[2006/04/18にブログで公開](https://nyanonon.hatenablog.com/entry/20060418/p2)したバージョン 0.4と同じものです。後継となる [asc](https://github.com/nyanonon/asc) を公開しています。

## 概要

ビジュアルアーツ系のゲームで使われているAVG32のシナリオデータをNScripter用に変換します。

NScripter  
http://www.nscripter.com/

ゲームとして必要な分岐や変数などは全く変換されません。  
あくまでテキストや画像、音声などを楽しむためだけに作成しています。

## 準備

AVG32で動作するゲームがインストールされているフォルダにある DATフォルダなどから、SEEN.TXT を探し出して下記のツールで分割します。

AVG32シナリオファイル解凍・分割ツール  
ビジュアルアーツのAVG32システム用／LittleNoiseさん提供  
http://hp.vector.co.jp/authors/VA028184/

SEEN001.TXT、SEEN002.TXT…のように分割されたシナリオデータから、NScripter用に変換したいものを選び、バイナリエディタなどで中を覗いてシナリオデータの開始アドレス（16進数）を調べます。判らなければ、開始アドレスは 0 でもかまいません。

DANDP Binary Editor（参考）  
http://hp.vector.co.jp/authors/VA020296/soft/dandp.htm

続いて、画像データを抽出します。BMPファイルに変換してください。

VisualArts系PDT/G00/MCG形式用Susie32プラグイン(透明色対応仕様)  
http://homepage3.nifty.com/hiro-inoue/soft_spi0.html#SOFT_SPIKNN

続いて、音声データを抽出します。avg32tools の koeunpac.exe を使って WAV ファイルに変換してください。

avg32tools  
http://www.geocities.co.jp/Playtown-Domino/8282/avg32tools_readme.html

## 動作

例としてSEEN124.TXTのシナリオデータ開始アドレスが00000B8Bだった場合、下記のように入力することで、NScripter 用シナリオデータが 0.txt として出力されます。

```
AVG2NSC SEEN124.TXT B8B > 0.txt
```

尚、SEEN124.TXT は AVG2NSC.EXE と同じフォルダに用意しておいてください。

あとは 0.txt と同じフォルダに BMP フォルダ、WAV フォルダを作成しておき、この中に画像データ、音声データをそれぞれ格納して、NScripter（nscr.exe）本体を置いて実行すればシナリオを実際に楽しむことができます。

## 動作についての注意

画像データ、音声データ以外に、ゲームの DAT フォルダや BGM フォルダに含まれる効果音や BGM の WAV ファイルがある場合は、これも WAV フォルダに入れておいてください。

## 仕様

変換できないAVG32の命令は「; error 」で始まるコメントとして、16進数や文字列のデータとして出力されています。  
このデータが何の意味を持つか判れば、さらに良い変換を行うことが出来るようになるかも知れません。

また、出力される NScripter 用シナリオデータ（例: 0.txt）は特定のゲーム用にいくつか勝手なパラメータで環境設定を行っています。  
NScripter で実行中にエラーとなる場合は、シナリオデータをメモ帳などで開いて、6行目の setwindow を削除するなど適宜パラメータを変更してください。

個人的に特定のゲームの特定のシナリオを楽しむためだけに作ったので、それ以外でどのような動作になるかは判りません。  
また、テキストに含まれる＊ＡＢ、＊ＢＢ、＊ＺＢは直也、真純、淳子に勝手に置き換えられます。  
本来はゲームがインストールされたフォルダにある GAMEEXE.INI の #NAME.ABなどの記述を参照して置き換えるべきものですが、面倒だったので勝手に決めうちで変換しています。

あと、シナリオ末尾に NScripter を終了させるための end 命令が入ってないので最後はエラーとなります。

## NScripter について

今回、こんなものを作るにあたって NScripter の仕様を色々調べました。
いわゆるプログラムにおける Hello World. は下記のようにテキストファイル 0.txt を作成して、同じフォルダに配置した nscr.exe を実行するだけで表示されます。

```
*define
game

*start
ハローワールド\
end
```

ちなみに *define と続く game は必須。game の部分で *start に飛びますので、*start も必須。end で NScripter を自動的に終了させられます。  

ハローワールドの末尾の「\」は、いわゆるクリック待ちです。続く表示で改ページさせたくない場合は「@」を使います。  
あとは改行すれば、そのまま改行となるし、空行が欲しければ「br」を記述します。

+ 画像は「bg "ファイル名",1」で表示。末尾の1は即時表示の意味です。
+ BGMは「bgm "ファイル名"」で演奏開始。「stop」で演奏終了。
+ 音声は「dwave 0,"ファイル名"」で再生。最初の0はチャンネル番号です。
+ 音声に重ねて効果音を鳴らしたい場合はチャンネル番号をずらします。

その他、詳しくは NScripter 公式ページを参照してください。

## ソースについて

開発はLSI-C86試食版で行っています。stdio 及び stdlib などの標準ライブラリしか使っていませんので、UNIX系のCコンパイラなどでも問題なくコンパイルできると思います。

## 参考：AVG32 命令（コマンド）一覧表

+ 01					クリック待ち
+ 02 03					改行
+ 0B 05 "画像" 00 xx			PDFフォルダ内の画像.PDTを表示（xxはエフェクト番号）
+ 0B 22 01 "画像1" 00 xx 01 "画像2" 00	PDFフォルダ内の画像1.PDT上に画像2.PDTを表示（xx はエフェクト番号）
+ 0E 01 "BGM名" 00			BGMフォルダ内のBGM名.WAVをループ再生
+ 0E 10					BGM再生停止（フェードアウト）
+ 0E 11					BGM再生停止（即時）
+ 0E 21 xx xx xx xx			KOEフォルダ内のZxxx.KOEの音声を再生（xx はKOE番号）
+ 0E 30 "SE名" 00				DATフォルダ内のSE名.WAVを再生
+ 13 01 xx				黒で画面クリア（xx はエフェクト番号）
+ 13 10 xx				白で即時画面クリア（xx はエフェクト番号？）
+ 17 01 xx				画面を揺らす（xx は効果番号）
+ 19 01 xx xx				一定時間待つ（xx xx は待ち時間）
+ 58 xx … "テキスト" … "テキスト" …	選択肢表示（詳細不明）
+ FF xx xx xx xx "テキスト" 00		テキスト表示（xx はテキスト番号）

※1 エフェクト番号 10、11、1B は 0.5 秒程度のクロスフェードと思われる。（GAMEEXE.INI の FADE で指定？）
※2 KOE番号は 4E 58 D3 0B だった場合、最初の 4 を無視した BD358E を 10進数に直した 124000014 となり、
    Z124.KOE から抽出した場合の Z124000014.WAV が該当する。
※3 効果番号 17 は上下に16ドット揺らす 0.5 秒程度の効果と思われる。（GAMEEXE.INI の SHAKE で指定？）
※4 待ち時間 20 7D は、1 秒程度だと思われる。
※5 テキスト番号は 0 から 1 ずつ、連番で付けられている。
※6 テキストが【】で囲まれる場合は、人物名として扱われ、【】は表示しない。
※7 テキスト中に＊で始まるアルファベット２文字は、GAMEEXE.INI の #NAME.AB="三沢" のように対応する
    名前に置き換えられる。
※8 テキストが「」で囲まれる場合は、折り返し時に"「"を避けて改行表示する。
例）
「あそぼーよ。今度はうまく出来るから。」
